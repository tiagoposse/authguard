# entapi integration example

This example shows how to integrate entauth with [Ent](https://entgo.io) and
[entapi](https://github.com/tiagoposse/entapi) for a complete auth system with:

- JWT login/register/refresh endpoints
- API key CRUD with per-key scope restrictions
- Session management with per-device revocation
- Guard annotations on entity schemas
- Custom entity-level ownership guards
- Admin elevation flow

## Structure

```
ent/
  entc.go                    # Code generation config (entoas + entapi + guards)
  schema/
    user.go                  # User schema with password hash, role, etc.
    apikey.go                # API key schema (persisted keys with metadata)
    session.go               # Session schema (device tracking + refresh tokens)
    article.go               # Example entity with guard annotations
internal/
  api/
    server.go                # Server setup, middleware stack, custom handlers
    auth.go                  # Login, register, refresh, logout handlers
    apikeys.go               # API key create/revoke handlers
    guards.go                # Custom guard functions (ownership checks)
```

## How It Works

1. **Code generation** (`ent/entc.go`): entapi reads `Guards()` annotations
   from schemas and injects auth checks into generated CRUD handlers via
   `AuthGuardHook`.

2. **Middleware** (`internal/api/server.go`): Bearer tokens, API keys, and
   admin elevation tokens are validated and injected into request context
   before any handler runs.

3. **Guard enforcement**: Generated handlers call `auth.Resolve("guardName")`
   for simple guards and inline entity-fetch code for ownership guards.

4. **Custom auth endpoints**: Login, register, refresh, and logout are
   hand-written (not generated) since they have unique business logic.

5. **API keys**: Stored in the database with metadata (name, scopes, last
   used). The actual key token is generated by entauth and returned once at
   creation time.

6. **Sessions**: Stored in the database for visibility (list active sessions,
   revoke by device). Token revocation uses entauth's in-memory
   `RevocationStore` for fast checks.
